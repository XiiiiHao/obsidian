# 数制与编码
十进制转换为任意进制：整数部分：除基取余；小数部分：乘基取整

### 编码表示
1. 原码：最高位表示符号，其余各位表示数的绝对值
$$
[x]_{原} = \begin{cases}0,x, \qquad \qquad 0\leq x<2^{n-1} \\ \\
2^{n-1}-x = 2^{n-1}+|x|, \qquad -2^{n-1}<x \leq 0
\end{cases} \qquad (x是真值)，字长为n
$$
	表示范围：$\displaystyle -(2^{n-1}-1)\leq x \leq 2^{n-1}-1$
	优点：与真值的转换简单；实现乘除运算比较方便
	缺点：正负0；实现加减运算比较复杂
2. 补码
$$
[x]_{补} = \begin{cases}0,x, \qquad \qquad 0\leq x<2^{n-1} \\ \\
2^{n}+x = 2^{n}-|x|, \qquad -2^{n-1} \leq x < 0
\end{cases} \qquad (x是真值)，字长为n
$$
	表示范围：$\displaystyle -2^{n-1} \leq x \leq 2^{n-1}-1$,比原码多表示一个$\displaystyle -2^{n-1}$
	Max = 0,11……1；Min = 1,00……0
3. 反码
	0的表示不唯一；范围比补码少一个最小负数
4. 移码
$$
[x]_{移} = 2^{n-1}+x \qquad (-2^{n-1} \leq x < 2^{n-1})
$$
	0的表示唯一：10000
	==一个真值的移码与补码仅差一个符号位==
	MAX = 11111(真值2^n-1)；MIN = 0000（真值：-2^n）

### 整数的表示
Unsigned
	$\displaystyle B 2 U(X)= \sum^{n-1}_{i=0}x_{i} \cdot 2^{i}$
	$\displaystyle UMin \quad = \quad 0 (000……0);\qquad UMax \quad = 2^{n}-1(1111……1)$
Two's Complements
	$\displaystyle B 2 T(X) = -x_{n-1} \cdot 2^{n-1}+\sum^{n-2}_{i=0}x_{i} \cdot 2^{i}$
	$\displaystyle TMin \quad = -2^{n-1}(1000……0); \qquad TMax = 2^{n-1}-1(011……1)$
	Minus 1 = 111111
==无符号与有符号运算自动转换为无符号==
零扩展与符号扩展

# 运算方法与运算电路
1. 一位全加器
	和表达式：$\displaystyle S_{i} = A_{i}\oplus B_{i} \oplus C_{i-1}$
	进位表达式：$\displaystyle C_{i} = A_{i}B_{i}+(A_{i} \oplus B_{i})C_{i-1}$
2. 串行进位加法器
	最长运行时间由进位信号传递时间决定
3. 并行进位加法器
4. 带标志加法器
	溢出标志$\displaystyle OF=C_{n} \oplus C_{n-1}$:用于判断==有符号==加法运算是否溢出
	符号标志$\displaystyle SF=F_{n-1}$:用于表示==有符号==数加法运算结果的正负
	零标志符$\displaystyle ZF$:位或取反
	进位/借位标志$\displaystyle CF = C_{out} \oplus C_{in}$，==CF=1时溢出，并且在进行无符号运算==
5. 算术逻辑单元ALU
	组合逻辑电路、核心是带标志加法器

### 定点数的移位运算
1. 逻辑移位
	逻辑左移若高位1移出则发生溢出
2. 算术移位
	若移出的高位不同于移位后的符号位，则发生溢出

### 定点数的加法运算
1. 补码加法
2. 溢出判别方法：符号相同相加或相异相减才会溢出
	(1)一位符号位：$\displaystyle V=A_{s}B_{s}\bar{S_{s}}+\bar{A_{s}}\bar{B_{s}}S_{s}$；参加运算的两个数符号相同，结果又跟原操作数符号不同
	(2)双符号位：模4补码，$\displaystyle V = S_{s1} \oplus S_{s2}$；两个符号位不同表示溢出
	(3)根据进位情况判断：$\displaystyle V=C_{n} \oplus C_{n-1}$；最高位进位与次高位进位不同表示有溢出
3. 加减法运算电路
> [!key]+ CF标志位的作用
> (1)无符号数大小的比较：ZF=1,A=B;ZF=0、CF=0,A>B; CF=1,A<B;
> (2)有符号数大小的比较：ZF=1,A=B;OF=SF且ZF=0,A>B;OF！=SF且ZF=0,A<B;
4. 原码的加减法

### 定点数的乘除运算
1. 乘法
	有符号：==高32位全部相等，等于低位符号表示不溢出==
	无符号：==高32位全部为0表示不溢出==
	符号位不参加运算，由两个数的符号位异或得到
2. 除法
	只有$\displaystyle -2^{n-1} / -1$会溢出，其余情况都不会溢出

# 浮点数
$$
V=(-1)^{S } \times M \times R^{E}
$$
### Normalized Values
* COndition : $\displaystyle exp \neq 0000  \quad and \quad exp \neq 1111$
* E = exp - Bias
	* exp:unsigned value of exp filed
	* Bias = $\displaystyle 2^{k-1}-1$
* M = 1.xxxx……x
	* xxxxxx: bits of frac filed
	* Minimun when frac = 0000(M=1.0)
	* Maximum when frac = 1111(M=2.0-$\displaystyle \epsilon$)
	* get extra leading bit for free

### Denormalized Values
* Condition : exp = 0000
* E = 1-Bias
* M = 0.xxxx

### Special Values
* condition :exp = 1111
* Case : exp = 1111,frac = 0000
	* represents value infinity $\displaystyle \infty$
* Case : exp =1111, frac $\displaystyle \neq$ 0000
	* Not a Number(NaN)

### IEEE 754标准
32位单精度：1+8+23
64位双精度：1+11+52
偏置值：$\displaystyle 2^{n-1}-1$;127和1023
真值:
$$
(-1)^{s} \times 1.f \times 2^{e-bias}
$$
最小值：e=1,f=0
最大值：e=254/2046,f=.11111
1)全0阶码全0尾数：正负0；0的符号取决于s
2）全1exp全0f：正负无穷
3）全1exp非0f：NaN
4）全0exp非0f：未规格化数
==规格化后阶码超出范围才发生溢出==
阶码上溢为溢出，阶码下溢出可用非规格化数表示，尾数上下溢出可用左移右移解决
> [!key]+ 强制类型转换
> int转为float，虽然不会溢出，但是尾数位数有限制，可能会有数据舍入，转为double则能保留精度
> float、double转为int,小数部分被截断，int表示范围更小还可能发生溢出
