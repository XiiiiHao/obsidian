# 指令系统
### ISA（指令集体系结构）
ISA规定：
1. 指令格式、指令寻址方式、操作类型，以及每种操作对应的操作数的规定
2. 操作数的类型、操作数寻址方式、大端or小端存储
3. 程序可访问的寄存器编号、个数和位数存储空间的大小和编址方式
4. 指令执行过程的控制方式，包括PC、条件码定义
### 指令的基本格式
	操作码+地址码字段
1. 0地址指令
	OP,仅用在堆栈计算机
2. 一地址指令
	$\displaystyle OP(A_{1}) \rightarrow A_{1}$
3. 二地址指令
	$\displaystyle (A_{1})OP(A_{2}) \rightarrow A_{1}$
4. 三地址指令
	$\displaystyle (A_{1})OP(A_{2}) \rightarrow A_{3}$
5. 四地址指令
	$\displaystyle (A_{1})OP(A_{2}) \rightarrow A_{3}，A_{4}为下条指令的地址$
### 指令的操作类型
* 数据传送：MOV、LOAD、STORE、PUSH、POP
* 算术与逻辑运算
* 移位操作
* ==转移操作==
	调用指令：返回主程序执行
	转移指令：不返回执行
* IO操作

# 指令的寻址方式
### 指令寻址
* 顺序寻址：PC+“1”；
* 跳跃寻址
	CPU总是从PC取指令
### ==数据寻址==
	操作码+寻址特征（方式）+形式地址A
1. 隐含寻址
	优点：缩短指令字长；缺点：需增加存储操作数或者隐含地址的硬件
2. 立即数寻址
	A就是操作数，补码表示
	优点：执行阶段==不访存==，指令执行速度最快；缺点：A的位数限制了立即数的范围
3. 直接寻址
	A就是操作数的真实地址，EA=A
	优点：简单不需要计算地址，只访存一次；缺点：A限制了操作数的寻址范围，操作数的地址不易修改
4. 间接寻址
	A是操作数的地址的地址，EA=（A）
	优点：可扩大寻址范围，便于编制程序（==方便子程序返回==）；缺点：多次访存，速度慢
5. 寄存器寻址
	$\displaystyle EA=R_{i}$
	优点：不访存改为访问寄存器，速度快，==寄存器数量远小于内存单元数，所以地址码位数少==，指令字长短
	缺点：寄存器价格昂贵，数量有限
6. 寄存器间接寻址
	$\displaystyle EA=(R_{i})$
	优点：既扩大寻址范围，又减少访存次数
7. 相对寻址
	$\displaystyle EA=(PC)+A$，PC自动自增
	==用于转移指令==
8. 基址寻址
	$\displaystyle EA=(BR)+A$
	可扩大寻址范围（BR位数大于A），有利于==多道程序设计，并发运行==，可用于编制浮动程序
9. 变址寻址
	$\displaystyle EA=(IX)+A$
	可扩大寻址范围（IX位数大于A），==适合编制循环程序==
10. 堆栈寻址
	软堆栈、硬堆栈（寄存器）
	访存次数：软1硬0

# CISC与RISC的基本概念
CISC：复杂指令系统计算机
* 指令系统庞大，数目一般在200条以上
* 指令长度不固定，指令格式多，寻址方式多

RISC：精简指令系统计算机
* 定长指令
* 只用LOAD/STORE指令访存
* 一定采用指令流水线技术，大部分指令在一个周期内完成
* 以硬布线为主，不用或少用微程序控制
