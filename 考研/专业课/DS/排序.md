# 内部排序
## 插入排序
### 直接插入排序

* 基本思想（边移边找）
	* 查找到插入位置k
	* L[k...i-1]依次后移
	* L[i]复制到L[k] (使用L[0]作为哨兵)```
* 性能分析
	* 空间效率：O(1)
	* 时间效率
		* 最好：O(n),每次只用比较一次不用移动元素,比较$\displaystyle n-1=\sum^{n}_{i=2}1次$
		* 最差：O(n^2)，逆序，比较$\displaystyle \frac{n(n-1)}{2}=\sum^{n}_{i=2}i-1$
		* 平均：O(n^2)
	* 稳定性：从后往前先比较再移动,为稳定排序
	* 适用性：顺序存储、链式存储的线性表
### 折半插入排序
* 基本思想
	* 先查找再移动
* 性能分析
	* 时间复杂度$\displaystyle O(n^{2})$，仅减少元素比较次数$\displaystyle O(n\log n)$，移动次数未变
	* 稳定
	* 仅适用于顺序存储的线性表
### 希尔排序
* 基本思想
	* 分组插入排序
* 性能分析
	* 时间复杂度：最坏$\displaystyle O(n^2)$，n在某个特定范围内$\displaystyle O(n^{1.3})$
	* 空间复杂度：$\displaystyle O(1)$
	* 稳定性：不稳定
	* 适用性：仅适用于顺序存储的线性表
## 交换排序
### 冒泡排序
* 基本思想
	* 把最大的换到最右边，边比较边换
	* 设置交换flag，没有交换操作说明有序，直接跳出循环
* 性能分析
	* 时间复杂度：
		* 最好情况：$\displaystyle O(n)$, 比较n-1次，移动0次
		* 最坏情况：逆序，比较$\displaystyle \frac{n(n-1)}{2}$次，每次比较要移动3次来交换位置
	* 空间复杂度：$\displaystyle O(1)$
	* 稳定性：稳定
	* 适用性：顺序存储和链式存储的顺序表
### 快速排序
* 基本思想
	* 分治，递归
* 性能分析
	* 时间复杂度：划分越对称越快
		* 最坏：基本有序或基本逆序，$\displaystyle O(n^{2})$
		* 最好：每次均衡划分，$\displaystyle O(n\log_{2}n)$，比较次数一点一点算
		* 快排平均情况更接近最佳情况，所有内部排序算法平均性能最优
	* 空间复杂度：借用工作栈
		* 最好：$\displaystyle O(\log_{2}n))$
		* 最坏：n-1次递归调用，$\displaystyle O(n)$
		* 平均$\displaystyle O(\log_{2}n)$
	* 稳定性：不稳定
	*  不产生有序子序列，但每次pivot都被放在正确位置上
	* 适用性：顺序存储的线性表
## 选择排序
### 简单选择排序
* 基本思想
	* 先确定最小元素位置再换
* 性能分析
	* 时间复杂度：$\displaystyle O(n^{2})$
		* 比较次数:与初始序列无关，$\displaystyle \frac{n(n-1)}{2}$
		* 移动次数: 最少0次，最多3（n-1）次
	* 空间复杂度：$\displaystyle O(1)$
	* 稳定性：不稳定
	* 适用性：顺序存储和链式存储的线性表，以及关键字较少的情况
### 堆排序(完全二叉树)
* 基本思想
	* 建堆，调整堆，建堆时反复调整堆
	* 用顺序存储在顺序表中的完全二叉树作为堆
	* 跟兄弟结点比较和跟子孙结点比较
	* 删除时，把最后一个加到堆顶，然后向下调整堆
	* 插入时，加到最后一个地方，然后对这个新结点进行调整
* 性能分析
	* 时间复杂度：建堆不超过4n，为$\displaystyle O(n)$,n-1次向下调整，每次调整O(h),最好最坏平均都为O(logn)，堆排序最好最坏都是O(nlogn)，==区分调整跟初始的复杂度==
	* 空间复杂度：$\displaystyle O(1)$
	* 稳定性：不稳定
	* 适用性：顺序存储的线性表
## 归并排序
* 基本思想
	* 二分递归排序，排完合并
* 性能分析
	* 时间复杂度：$\displaystyle O(n\log n)$，每趟归并要调用$\displaystyle \lceil \frac{n}{2h} \rceil次 Merge()函数$ $\displaystyle O(n)$,$\displaystyle \lceil\log_{2} n \rceil$次归并（向上取整）
	* 空间复杂度：$\displaystyle O(n)$，使用了辅助数组来合并
	* 稳定性：稳定
	* 适用性：适用于顺序存储和链式存储的线性表

## 基数排序
* 基本思想
	* 按关键字分配，然后收集，换关键字重复
* 特点
	* 不基于移动和比较
	* ==不能对float,double类型排序==
* 性能分析
	* 时间复杂度：$\displaystyle O(d(n+r))$
		* d趟分配和收集
		* 每次分配都遍历所有关键字，$\displaystyle O(n)$
		* 每次收集合并r个队列$\displaystyle O(r)$
	* 空间复杂度：r个队列，r个队头指针和队尾指针，O(r)
	* 稳定性：稳定，每次从前往后收集
	* 适用性：顺序存储和链式存储的顺序表

# 外部排序
### 多路归并排序
k路归并排序要在内存中分配k个输入缓冲区，和1个输出缓冲区
时间开销
	读写外存的时间+内部排序所需的时间+内部归并时间
优化：
	增加归并路数k,代价：增加输入缓冲区；归并时需要k-1次关键字比较
	减少初始归并段数量r即增加其长度

### 败者树
用于优化k增大时带来的代价
第一次构造败者树需要对比$\displaystyle k-1次关键字$，然后选出最小元素只用对比$\displaystyle \lceil \log_{2}k \rceil$次
