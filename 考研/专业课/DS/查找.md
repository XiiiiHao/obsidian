


# 线性查找
### 顺序查找
* 一般线性表的查找
	* $\displaystyle ASL_{成功}= \sum^{n}_{i=1}P_{i}(n-i+1)= \frac{n+1}{2}$
	* $\displaystyle ASL_{失败} = n+1$
	* 缺点：n较大时平均查找长度较大，效率低
	* 优点：对存储和有序性没有要求
* 有序线性表的查找
	* 判定树上n个结点，n+1个失败结点
	* $\displaystyle ASL_{成功}= \sum^{n}_{i=1}P_{i}(n-i+1)= \frac{n+1}{2}$
	* $\displaystyle ASL_{失败}= \frac{1+2+3+ \dots+n+n}{n+1} = \frac{n}{2}+\frac{n}{n+1} \qquad(失败结点上一个结点的层数相加)$
###  折半查找
* 判定树为平衡二叉树
* 查找给定值不会超过树的高度$\displaystyle [\log_{2}(n+1)]或[\log_{2}n]+1$
* $ASL_{成功}＝$
* 查找不存在值最多次数为树的高度h，最少为h-1
* 仅适合顺序存储，要有随即存取特性
### 分块（索引顺序）查找
* $\displaystyle ASL = L_{I}+L_{S} \qquad(索引查找+块内查找)$
* $\displaystyle ASL = L_{I}+L_{S}=\frac{b+1}{2}+\frac{s+1}{2}=\frac{s^{2}+2s+n}{2s}$
* $(n,均分为b块，每块s个记录，等概率，顺序查找，s=\sqrt{ n }时平均查找长度最小为\sqrt{ n }+1)$

# 树形查找
### 二叉排序树（BST）
* 定义（3条特性）
* 查找、插入、构造、删除的基本思想
* 删除
	* 叶子结点直接删掉
	* 只有单一左子树或者右子树，直接顶替
	* ==找前驱：左子树的最右下（即左子树里最大的）；直接后继：右子树里最小的==
* 效率分析
	* 平均查找长度
		* 若为平衡二叉树，则为$\displaystyle \log_{2}n$
		* 若为单支树(输入序列有序)则退化为$\displaystyle O(n)$
	* 相同的关键字插入顺序不同可能构成不同的二叉排序树

### 平衡二叉树 AVL
平衡因子：左子树高度-右子树
插入操作：
	1.LL平衡旋转（右单旋转）
	2.RR平衡旋转（左单旋转）
	3.LR平衡旋转（先左后右）
	4.RL
> [!solution]+ 深度为h的平衡树中含有的最少结点数
> $\displaystyle n_{h}=n_{h-1}+n_{h-2}+1$
> $\displaystyle n_{0}=0,n_{1}=1,n_{2}=2,n_{3}=4,n_{4}=7$

删除操作：
	1.删除结点
	2.一路向北找到最小不平衡子树，找不到直接完结撒花
	3.找到这个不平衡子树的个头最高（==看谁下面子树高==）的儿子，孙子确定调整方式
	4.如果不平衡向上传导，继续2
### 红黑树
性质：
	1.结点要么红要么黑
	2.根结点是黑色
	3.虚拟外部结点，失败结点为黑
	4.不存在相邻的两个红结点
	5.对于每个结点，到虚拟外部结点失败结点的简单路径上，包含的黑色结点数量相同
左根右，根叶黑，不红红，黑路同
**结论1**：从根结点到叶子结点的最长路径不超过最短路径的2倍
**结论2**：n个内部结点，$\displaystyle h\leq 2\log_{2}(n+1)$
#### 插入
黑叔：旋转（跟AVL一样的旋转方式）+染色（LL,RR父换爷染色，LR,RL儿换爷染色）
红叔：染色（叔父爷染色）+变新（爷结点看作新插入的结点，根黑非根红）
> [!solution]+ 黑高为h的RBT,内部结点数量最少是
>满黑树，$\displaystyle 2^{h}-1$

                                ### B树
m阶B树：平衡因子均为0的m路平衡查找树
性质：
	1.每个结点最多有m棵子树，至多m-1个关键字
	2.若根结点不是叶结点，则至少有2棵子树，
	3.除根结点外所有非叶结点至少有\[m/2\]棵子树，向上取整
	

> [!solution]+ 选择题计算
> 1.n个关键字的m阶B树最小高度，最大高度为
> 	最小高度：每个结点m-1个关键字，m个分叉，$\displaystyle h\geq \log_{m}(n+1)$
> 	最大高度：分叉尽可能少，根结点2叉，其他$\displaystyle \lceil \frac{m}{2} \rceil$,h+1层共有叶子结点$\displaystyle 2\lceil \frac{m}{2} \rceil^{h-1}$个，必有n+1个叶子结点，大于前面那个值，$\displaystyle h\leq \log_{\left\lceil  \frac{m}{2} \right\rceil} \frac{n+1}{2} +1$

插入：
	定位，如果多了，就把最中间的往父节点提（提到当前指针的右边），左右的分裂成两个，父节点也超过上限则继续分裂
删除：
	终端节点：
		1.直接删除：关键字满足最少的要求，则直接删掉
		2.兄弟够借：右够借用后继和后继的后继依次顶替空缺，左够借用前驱与前驱的前驱顶替
		3.左右兄弟都不够借：父节点+左右兄弟合并
	非终端节点：
		直接前驱：左边最大的
		直接后继：右边最小的
### B+树
条件：
	1.每个分支最多m棵子树
	2.非叶根结点至少两棵子树，其他分支结点至少$\displaystyle \lceil m\\/2 \rceil$
	3.结点的子树与关键字个数相等
	4.叶结点包含==关键字==与指向记录的==指针==，关键字按大小顺序排列，相邻叶子结点连接起来，==支持顺序查找==
	5.所有分支结点仅包含子节点中==关键字最大的值==与指向子节点的==指针==
> [!solution]+ B树与B+树的差异
>1.B+树，n个关键字n棵子树，B树：n个关键字n+1棵子树

# 散列表
### 散列函数（开放定址法，拉链法）                                          
* 直接定址法：$\displaystyle H(key)=a \times key+b$，适用于基本连续
* 除留余数法：$\displaystyle H(key)=key\%p$  p取最接近m(散列表表长)的质数
* 数字分析法
* 平方取中法
### 处理冲突的方法
* 开放定址法：$\displaystyle H_{i}=(H(key)+d_{i})\%m$
	* 线性探测法：$\displaystyle d_{i}=1,2,3,4\dots,m-1$
	* 平方探测法：$\displaystyle d_{i}=1^{2},-1^{2},\dots k^{2},-k^{2},k\leq \frac{m}{2}$  m为能表示成4k+3的质数
	* 双散列法
	* 伪随机序列法
	* 删除是逻辑删除
* 链接法：==无聚集现象==
### 查找与性能分析
* 查找效率取决于三个因素：散列函数、处理冲突的方法、装填因子
* 平均查找长度依赖于装填因子，不直接依赖于表中记录数n和散列表长度m

$\displaystyle ASL_{成功} = \frac{1}{n},n为存在个元素个数$
$\displaystyle ASL_{失败}= \frac{1}{r},r为散列函数取值的个数$
为空算一次查找，逻辑删除的要继续往后查找到下一个为空
