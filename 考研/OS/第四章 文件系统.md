# 文件的管理
* 文件控制块：文件名、文件类型、文件结构、文件的物理位置、存取控制信息（存取权限）、管理信息
* 索引节点：文件属性信息构成的数据结构又称i节点，此时文件目录项仅由文件名和索引节点号构成
* 文件的操作
	* 建立文件：分配外存、在目录中新建目录项、记录属性到目录项里面
	* 删除文件：在目录中找到待删除目录项使之成为空闲目录项、回收储存空间
	* 读文件：查找目录找到目录项、从目录项得到在外存的地址、从外存将数据读入内存
	* 写文件：查找目录找到目录项，利用目录中的文件指针将信息写入文件
	* 打开文件：将待访问文件的目录信息读入内存打开文件表中，建立起用户进程和文件之间的联系
	* 关闭文件：撤销主存中有关该文件的目录信息，切断用户进程与文件的联系；若被修改，则写回辅存
* 目录的操作
* 常见目录结构
	* 单级目录结构：易于实现管理简单；文件多查找时间长，易重名
	* 二级目录结构：主文件目录、用户文件目录，缺乏灵活性、无法反映真实世界
	* 多级目录结构：树形
	* 图形目录结构：unix系统里面的链接
* 目录查询技术

# 磁盘存储管理
* 文件读写时间
	* 寻道时间（==所占比例最大==）：磁头从当前位置到指定磁道所经历的时间，有启动磁臂时间跟移动时间构成
	* 旋转延迟时间：扇区移动到磁头下面经历的时间，平均是每转所需时间的一半
	* 传输时间：从磁盘上读出数据或向磁盘写入数据所经历的时间
* 磁盘寻道算法
	* 先来先服务FCFS：
	* 最短寻道时间优先SSTF：寻道性能好，饥饿，不能保证平均寻道时间最短
	* 扫描算法SCAN：在当前移动方向上选择最近的作为下一次服务的对象，寻道性能好，避免饥饿，不利于两端磁道的请求
	* 循环扫描算法CSCAN：磁头单向移动，循环
	* N-step-SCAN：分成若干子队列，按FCFS依次处理子队列，处理每个队列时按SCAN算法进行
	* FSCAN：上面的简化，只分两个子队列
* 空闲空间管理
	* 空闲文件目录
	* 空闲块链
	* 位示图法
* 磁盘容错技术
	* 第一级容错技术（低级）：主要防止因磁盘缺陷造成的数据丢失
		* 双份目录、双份文件分配表
		* 热修复重定向与写后读校验
	* 第二级容错技术：防止由磁盘驱动器和磁盘控制器故障导致的故障
		* 磁盘镜像
		* 磁盘双工
	* 第三级容错技术
